     3) !    First subroutine. This subroutine is responsible for setting up the run so each node  !
     4) ! can work independently (or almost). This is the only subroutine that is called           !
     5) ! differently whether the node is a master or a slave (truly speaking, the model almost    !

    10) ! 1. ed_1st_master: run only by the master, it reads the namelist, the met drivers, and    !
    11) !    splits the polygons accross the nodes, keeping some to itself. Then it sends the      !
    12) !    namelist/met driver information to the slave nodes.                                   !
    13) ! 2. ed_1st_node: run only by the slaves, it gets the information from the master so the   !
    14) !    modules can be properly initialized.                                                  !
